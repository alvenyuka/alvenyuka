## What I Build

### 🔍 [Fraud Detection System](https://github.com/alvenyuka/Fraud-Detection-System)
`PaySim` · `Python` · `XGBoost` · `Random Forest` · `SHAP`

A production-grade fraud classifier for mobile money networks, trained on 6.36M synthetic transactions with a **time-based split** — because fraud models predict the future from the past and random splits leak future state. **99.11% precision at 96.88% recall** on a 132K held-out window, calibrated to Brier 0.0005. Permutation importance shows the engineered `errorBalanceOrig` feature carries ~85% of predictive power. The hard part wasn't the model — it was making the outputs explainable enough for a KYC/AML audit trail.

### 📈 [Stock Portfolio Tracker & Analytics Engine](https://github.com/alvenyuka/Stock-Portfolio-Tracker-Analytics-Engine)
`Excel 365` · `CAPM` · `VaR` · `CVaR` · `Sharpe` · `Sortino` · `Calmar` · `Black–Litterman`

An institutional-grade portfolio analytics platform built natively in Excel 365 — STOCKHISTORY, linked Stocks data type, dynamic arrays, no VBA. Covers 16 stocks across 10 industries over 7 years, with drawdown analysis, risk-adjusted return attribution, parametric / historical / Monte Carlo VaR, Cornish-Fisher VaR, tax-aware lot matching under IRS §1222, Black–Litterman optimisation, and a 23-test validation suite that has to PASS before the dashboard renders. CFA Level I–III methodology end-to-end.

### 🧮 [Financial Analyst Portfolio](https://github.com/alvenyuka/Financial-Analyst)
`Excel` · `Three-statement models` · `DCF` · `Equity research`

A growing collection of company financial analyses built from primary-source filings — three-statement models, DCF valuations, scenario toggles, and a validation tab that ties every historical line back to the filing it came from. First model: Apple Inc.
